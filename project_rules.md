## 4. White-Label Telegram Mini App — Архитектура

### 4.1 Концепт
Telegram Mini App — это WebApp, который открывается внутри Telegram через кнопку `web_app`.
Контент (услуги, цены, мастера, фото) загружается **динамически из Notion** через наш backend.

### 4.2 Схема взаимодействия

```
Клиент (Telegram)
       │
       ▼
[Кнопка 🌐 Открыть Каталог] ──► WebApp URL (mini-app/index.html)
                                         │
                          ┌──────────────┘
                          ▼
               [Mini App Frontend]
               React / Vanilla JS
               Telegram.WebApp.initData
                          │
                          ▼ POST /api/catalog
               [Backend: FastAPI / Flask]
               Проверяет initData (HMAC-SHA256)
               Запрашивает Notion API
                          │
                          ▼
               [notion_db.py / Notion API]
               Базы: Услуги, Мастера, Портфолио
                          │
                          ▼
               JSON → Frontend рендер
```

### 4.3 Передача initData (безопасность)
- Telegram передаёт `window.Telegram.WebApp.initData` при открытии WebApp
- Backend **обязан** верифицировать подпись: `HMAC-SHA256(data_check_string, SHA256(bot_token))`
- Только после верификации — запрос к Notion и возврат данных

### 4.4 Динамический рендер из Notion
Создать отдельные Notion-базы:
| База | Поля |
|---|---|
| `Услуги` | Название, Категория, Цена, Описание, Фото (files) |
| `Мастера` | Имя, Специализация, Фото, Рейтинг |
| `Портфолио` | Фото (files), Категория, Мастер, Описание |

Backend эндпоинты:
- `GET /api/catalog` — возвращает услуги + цены из Notion
- `GET /api/masters` — список мастеров с фото
- `GET /api/portfolio` — галерея работ
- `POST /api/book` — создаёт запись (проксирует в `save_booking`)
